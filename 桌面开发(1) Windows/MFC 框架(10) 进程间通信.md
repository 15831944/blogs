# 剪贴板 Clipboard

1. 同一台主机两个进程间通信

# 匿名管道 Anonymous Pipe

1. 匿名管道只能本机在父子进程之间进行通信，子进程只能通过从父进程继承来获取匿名管道的读写句柄(调用 CreateProcess 函数)，并利用此句柄和父进程进行通信

# 命名管道 Named Pipe

1. 命名管道是网络通信，建立客户端服务器通信体系，采用命名管道文件系统(Named Pipe File System, NPFS)接口
2. 不像socket编程，命名管道不需要编写身份认证部分代码
3. 只能有服务器建立命名管道并接手客户端的连接请求，客户端只能跟现有的命名管道服务器建立连接

通信模式

1. 字节模式，数据以一个连续的字节流的形式，在客户端和服务器之间流动
2. 消息模式，客户端和服务器通过一系列不连续的数据单位，进行数据的收发，每次在管道上发出一条消息后，它必须作为一条完整的消息读入

1. 同一个命名管道可以创建多个实例，一个管道实例只能供一个服务器和一个客户端通信

# 邮槽 Mail Slot

1. 邮槽是基于广播(一个客户端广播到多个服务器)通信体系设计的，采用无连接的不可靠的数据传输
2. 邮槽是一种单向通信机制，服务器创建邮槽并读取数据，客户端打开邮槽并写入数据
3. 为了 Windows 系统兼容性，消息长度应小于424字节，数据量较小
